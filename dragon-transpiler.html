Dragon Transpiler Project

The Dragon Transpiler Project is a spinoff of my failed Dragon Compiler Project.
Instead of creating a full compiler with frontend and backend, I want to create a transpiler from the Dragon Language to C. Maybe also to Java.
This has several Advantages:

  • not implement something which has already been implemented
  • build on top of existing ecosystem.
  • enable people to have efficient C code which can be modified
  • write most of the code at a higher level. But still be able to go down towards the lower level, without using a big language like C++ with it's many complexities.
C builds on top of Assembly Language. It is an abstraction thereof.
Why not build an even higher level Language on top of C?
Most Languages that are higher level than C are not implemented on top of it.
